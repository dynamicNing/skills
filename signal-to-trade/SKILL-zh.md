# 事件驱动投资同心圆

将"事件"视为结构性断裂过程，而非单纯标题。从事件核心建立四层同心圆推演图，输出可证伪的多空选择与风险边界。

## 第 1 步：事件判定

满足以下大多数条件时，才将输入视为事件：

- 现有秩序正在断裂���旧共识无法再解释定价行为）。
- 存在重新定价压力（估值锚、折现率或现金流路径正在改变）。
- 结构性路径发生转移（产业链、资本路由、政策框架或风险溢价）。
- 具备过程性特征（`异响 → 扩散 → 确认 → 共识`），而非一次性噪音。

若条件较弱，需明确说明："此事件更可能是信息噪音，而非结构性事件"——并列出尚缺少的证据。

## 第 2 步：四层同心圆推演

### Circle 0 — 事件核心
- 发生了什么，哪个旧假设已失效。
- 当前所处阶段：`异响 / 扩散 / 确认 / 共识`
- 率先变化的关键变量：政策 / 需求 / 供给 / 利率 / 流动性 / 监管 / 地缘政治 / 技术

### Circle +1 — 逻��方向
推断一阶传导路径，使用方向性语言：
- "提升/压低增长预期"
- "扩大/压缩利润率"
- "压缩/扩张风险溢价"
- "提前/推迟资本开支周期"
- "定价权向上游/下游转移"

### Circle +2 — 节点扩散
绘制方向扩散至哪些节点：

| 节点类型 | 示例 |
|---|---|
| 宏观节点 | 增长、通胀、实际利率、信用利差、汇率、流动性 |
| 行业节点 | 上游原材料、核心设备、软件/服务、分销渠道、终端需求行业 |
| 资本节点 | 被动/主动资金流、政策性资金、杠杆资金、对冲需求 |

每个节点需提供：影响符号（`++/+/0/-/--`）、时滞（`即时/1-3月/3-12月/12月+`）、置信度及关键验证数据点。

### Circle +3 — 可交易映射
将节点映射至各市场标的：A 股、港股、美股、ETF/指数、衍生品（股指期货、期权、商品期货、波动率产品、汇率代理工具）。

每个标的需包含：`方向`（Long/Short/Pair）、`逻辑桥`（节点 → 标的链路）、`催化`（可观测触发因素）、`失效条件`（证伪触发器）、`拥挤度`（低/中/高）。

## 第 3 步：概率/赔率纪律

对每个核心判断打分：
- `概率`（0–100%）
- `赔率`（预期上行/下行非对称性）
- `兑现时窗`
- `数据置信度`

至少提供三种情景：**基准情景**、**乐观情景**、**悲观情景**——并说明哪些证据会迫使观点更新。

## 第 4 步：信源搜索

涉及当前或快速变化的信息时，先搜索再下结论。

信源优先级：
1. 优先读取本地完整信源清单（如存在）：工作区根目录下的 `搜索信源-主题分组汇总.md`
2. 若不存在或不完整，使用备用分组：[references/source-groups.md](references/source-groups.md)
3. 优先使用一手信源（官方数据、监管机构、公司 IR/财报），再参考媒体/KOL/情绪。
4. 关键事实主张至少经两个独立信源交叉验证。
5. 附上信源清单，包含发布日期和直接链接。

## 第 5 步：标准化输出结构

始终按以下顺序输出：

1. **事件判定** — 是结构性事件还是噪音？
2. **同心圆推演** — Circle 0 → +1 → +2 → +3
3. **投资映射清单** — 表格格式（见下）
4. **情景树与动态更新条件** — 基准/乐观/悲观 + 触发观点更新的条件
5. **风险与反例** — 哪些因素会证伪判断
6. **信息源** — 含日期和链接

### 投资映射清单最小表格格式

| 市场 | 标的 | 方向 | 逻辑桥 | 催化 | 失效条件 | 概率 | 赔率 | 时窗 |
|---|---|---|---|---|---|---|---|---|
| A股/港股/美股/ETF/衍生品 | Ticker 或标的名称 | Long/Short/Pair | 节点 → 标的 | 可观测触发因素 | 明确证伪条件 | % | 高/中/低 | 时间窗口 |

## 质量与安全护栏

- 明确区分事实、推断与猜测。
- 不捏造股票代码、合约代码或政策细节。
- 不确定的映射标注为 `待验证`。
- 保持分析性、可证伪的语气；避免确定性预测。
- 声明输出为研究分析，非个性化投资建议。

- "Use $signal-to-trade: 国内地产政策边际转向是否构成事件？给我多空两套方案。"
